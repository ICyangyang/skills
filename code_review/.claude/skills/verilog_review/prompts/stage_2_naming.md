## 命名规范要点

1. **信号命名**: 小写字母加下划线（如 `xx2yy_din`, `clk`, `rst_n`）
2. **宏定义**: 全大写（如 `define WIDTH 8`）
3. **参数**: 全大写（如 `parameter DATA_WIDTH = 32`）
4. **多位总线**: 使用降序 `[N-1:0]`
5. **打拍寄存器**: 使用 `_r` 后缀
6. **上一周期信号**: 使用 `_d` 后缀
7. **低电平有效信号**: 使用 `_n` 后缀

## 模块例化规范

**正确的例化方式**:
```verilog
module_name u_module_name (
    .port1(signal1),
    .port2(signal2),
    .port3()  // 未使用的端口用空括号表示
);
```

**错误的例化方式**:
```verilog
// 位置端口映射（禁止）
module_name u_module_name (
    signal1, signal2, signal3
);

// 例化时进行逻辑操作（禁止）
module_name u_module_name (
    .port1(signal1 & signal2),  // 错误
    .port2(signal2)
);

// 缺少端口（禁止）
module_name u_module_name (
    .port1(signal1)
    // 缺少port2
);
```

## 检查步骤

1. 读取目标Verilog文件
2. 提取所有信号名、模块名、参数名、宏定义
3. 检查命名分隔符（是否使用下划线）
4. 检查字符集（是否只包含字母数字下划线）
5. 检查开头字符（是否为字母）
6. 检查是否存在大小写区分的情况
7. 检查多bit总线定义（是否使用降序）
8. 检查宏定义是否全大写
9. 检查所有模块例化：
   - 是否使用命名端口映射
   - 例化端口中是否进行逻辑操作
   - 是否缺少端口
   - IP例化的parameter处理
10. 记录所有检查结果
