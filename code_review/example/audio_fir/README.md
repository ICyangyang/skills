# Audio FIR 滤波器微架构设计文档

## 1. 模块设计目标与规格

### 1.1. 核心功能
本模块 (`audio_fir`) 旨在实现一个功能完备的音频专用有限脉冲响应（FIR）滤波器。其核心任务是对接收到的音频数据流进行数字滤波处理，通过可配置的滤波器系数实现各种音频处理效果，如低通、高通、带通等。模块设计了一个流水线架构，支持连续的音频数据处理。

### 1.2. 关键特性
- **可配置性**: 支持通过参数配置输入输出数据宽度（默认16位）和滤波器阶数（默认18阶）。
- **流水线架构**: 采用流水线处理方式，支持连续的数据输入和输出，提高数据吞吐率。
- **系数可配置**: 通过外部输入提供滤波器系数，允许动态调整滤波特性。
- **状态控制**: 提供 `start` 和 `enable` 信号，允许外部系统控制滤波器的启动和运行状态。
- **初始化机制**: 包含初始化计数器和完成信号，确保滤波器在开始处理前完成内部状态的预填充。
- **MAC单元接口**: 设计了专用接口连接乘加（MAC）单元，用于执行核心的滤波运算。

### 1.3. 性能与约束
- **时钟域**: 所有逻辑均同步于输入时钟 `clk` 的上升沿。
- **滤波器阶数**: 默认为18阶，可通过 `STAGE` 参数进行配置。
- **数据宽度**: 输入输出数据默认为16位，可通过 `DIN_WIDTH` 和 `DOUT_WIDTH` 参数进行配置。
- **数据有效性**: 通过 `din_vld` 和 `dout_vld` 信号控制数据的有效性。

## 2. 整体架构与模块划分

该设计在逻辑上由几个核心组件构成：主状态机、FIR FIFO缓冲器、MAC单元接口和控制逻辑。

```text
+-------------------------------------------------------------------------+
| audio_fir (Top Module)                                                  |
|                                                                         |
|  +-----------------------+      +--------------------------+           |
|  |                       |      |                          |           |
|  |   audio_fir_fifo      |----->|      Main Control        |<--------+ |
|  | (FIFO for input data) |      |      Logic & State       |         | |
|  |                       |      |      Machine             |         | |
|  +-----------------------+      +--------------------------+         | |
|         ^                                 |                           | |
|         | din, din_vld                    | fir_fifo                  | |
|         |                                 |                           | |
|         |                            +----v----+                      | |
|         |                            |  MAC    |                      | |
|         |                            |  Unit   |                      | |
|         +----------------------------+ Interface|                      | |
|                                      +----+----+                      | |
|                                           |                           | |
|                                           | mac_sum, mac_out          | |
|                                           |                           | |
|                                           v                           | |
|                                      +----+----+                      | |
|                                      |  Result |                      | |
|                                      | Buffer  |                      | |
|                                      +----+----+                      | |
|                                           |                           | |
|                                           v                           | |
|                                      +----+----+                      | |
|                                      | Output  |                      | |
|                                      | Buffer  |                      | |
|                                      +---------+                      | |
|                                           |                           | |
|                                           v                           | |
|                                      dout, dout_vld                   | |
|                                           |                           | |
+-------------------------------------------|---------------------------|---+
                                            |                           |
                                      串行输出接口                    外部控制接口
```

- **`audio_fir_fifo`**: 作为一个独立的FIFO缓冲器，负责输入音频数据的存储和管理，提供延迟线功能。
- **`Main Control Logic`**: 这是模块的控制核心，包含状态机和各种控制逻辑，负责协调整个滤波过程。
- **`MAC Interface`**: 与外部乘加单元的接口，负责数据的输入和结果的输出。

## 3. 接口信号详细规范

| 信号名称      | 位宽 | 方向 | 描述 |
|---------------|:----:|:----:|:---|
| `clk`         | 1    | 输入 | **系统主时钟**。所有时序逻辑都在此时钟的上升沿触发。 |
| `rstn`        | 1    | 输入 | **低电平有效异步复位**。当为0时，模块所有状态和寄存器被强制初始化。 |
| `din`         | 16   | 输入 | **输入音频数据**。有符号的16位音频采样数据。 |
| `din_vld`     | 1    | 输入 | **输入数据有效信号**。高电平表示 `din` 数据有效。 |
| `dout`        | 16   | 输出 | **输出滤波后音频数据**。有符号的16位滤波后音频采样数据。 |
| `dout_vld`    | 1    | 输出 | **输出数据有效信号**。高电平表示 `dout` 数据有效。 |
| `start`       | 1    | 输入 | **启动信号**。高电平有效，启动滤波过程。 |
| `enable`      | 1    | 输入 | **使能信号**。高电平有效，使能模块运行。 |
| `cnt`         | 5    | 输入 | **计数器**。用于FIFO选择和滤波器控制。 |
| `coef`        | 20   | 输入 | **滤波器系数**。有符号的20位滤波器系数。 |
| `mac_a`       | 16   | 输出 | **MAC单元输入A**。连接到MAC单元的输入A端口。 |
| `mac_b`       | 20   | 输出 | **MAC单元输入B**。连接到MAC单元的输入B端口。 |
| `mac_c`       | 20   | 输出 | **MAC单元输入C**。连接到MAC单元的输入C端口。 |
| `mac_sum`     | 20   | 输入 | **MAC单元结果**。MAC单元计算的中间结果。 |
| `mac_out`     | 16   | 输入 | **MAC单元输出**。MAC单元计算的最终结果。 |
| `init_done`   | 1    | 输出 | **初始化完成信号**。高电平表示模块初始化完成。 |

## 4. 内部微架构实现细节

### 4.1. `audio_fir_fifo` 子模块

- **存储结构**: 使用一个深度为18、位宽为16的FIFO，作为FIR滤波器的延迟线。
- **功能**: 将输入的音频数据延迟一定周期，以实现滤波器的抽头（tap）功能。
- **接口**:
    - `din`, `din_vld`: 输入数据和有效信号。
    - `sel`: 选择信号，用于选择延迟后的数据。
    - `dout`: 延迟后的数据输出。
    - `clk`, `rstn`: 时钟和复位信号。

### 4.2. 主状态机

- **状态列表**: `S_IDLE` (空闲), `S_FIR` (滤波处理)。
- **关键寄存器**:
    - `state`: 当前状态机状态。
    - `state_nxt`: 下一状态。
- **状态转换**:
    - **`S_IDLE`**: 模块空闲状态。等待 `start` 信号变高后，跳转到 `S_FIR` 状态。
    - **`S_FIR`**: 滤波处理状态。持续处理音频数据直到 `fir_done` 信号变高后，返回 `S_IDLE` 状态。
- **控制信号**:
    - `fir_done`: 滤波完成信号，当 `cnt` 等于 `STAGE-1` 时有效。

### 4.3. 初始化机制

- **计数器**: `init_cnt` 用于追踪初始化进度。
- **逻辑** (`init_cnt_temp` 信号):
    - 当 `enable` 为低时，计数器归零。
    - 当 `init_done` 为高时，计数器保持不变。
    - 当 `din_vld` 为高时，计数器递增。
- **完成信号**: `init_done` 当 `init_cnt` 等于 `STAGE` 时有效，表示初始化完成。

### 4.4. MAC单元接口

- **输入A**: `mac_a` 连接到 `fir_fifo` 的输出，提供延迟后的音频数据。
- **输入B**: `mac_b` 连接到 `coef`，提供滤波器系数。
- **输入C**: `mac_c` 连接到 `result`，提供前一次计算的结果。
- **结果处理**: `result_temp` 根据 `start` 信号决定是清零还是接收 `mac_sum`，实现滤波器的累加功能。
- **输出处理**: `dout_temp` 根据状态和 `fir_done` 信号决定是否更新输出数据。

## 5. 时序与波形示例

### 5.1. 初始化时序

```wavedrom
{ signal: [
  { name: 'clk',         wave: 'p...........' },
  { name: 'enable',      wave: '0.1.........' },
  { name: 'din_vld',     wave: '0.1.1.1.....' },
  { name: 'init_cnt',    wave: '2.3.4.5.2...', data: ['0', '1', '2', 'N', '0'] },
  { name: 'init_done',   wave: '0.........1.' },
]}
```
当 `enable` 变高后，随着 `din_vld` 有效，`init_cnt` 递增直到等于 `STAGE`，此时 `init_done` 变高。

### 5.2. 滤波处理流程

```wavedrom
{ signal: [
  { name: 'clk',          wave: 'p....................................' },
  { name: 'state',        wave: '2.3.2...............................', data: ['S_IDLE', 'S_FIR', 'S_IDLE'] },
  { name: 'start',        wave: '0.1.0...............................' },
  { name: 'fir_done',     wave: '0............1.0....................' },
  { name: 'dout_vld',     wave: '0............1.0....................' },
  { name: 'dout',         wave: 'x............3.x....................', data: ['Filtered Data'] },
]}
```
1.  在 `S_IDLE` 状态，`start` 信号变高后，状态切换到 `S_FIR`。
2.  在 `S_FIR` 状态，模块持续处理数据。
3.  当 `fir_done` 信号变高，表示当前滤波完成，`dout_vld` 变高，输出滤波结果。
4.  状态返回 `S_IDLE`。
