### 检查项38: 变量声明检查（多赋值表达式）
**规范要求**: 一行是否存在多个变量的赋值表达式。参考值：否。

**审查结果**: ❌ 未通过

**详细说明**:

**问题代码**:
```verilog
第15行: state<=0;d_out<=0;con<=0;
```

**问题分析**:
- 在同一行中包含3个赋值语句，用分号分隔
- 违反规范要求"禁止一行存在多个赋值表达式"
- 降低代码可读性，不利于调试和维护

**规范引用**:
根据 `Verilog_RTL_Coding_Standards-v1.4.3.md` 第4条第14款：
> 禁止一行存在多个赋值表达式。

**示例**:
```verilog
// 不规范写法（当前）
if (~res) begin
    state<=0;d_out<=0;con<=0;
end

// 规范写法
if (~res) begin
    state <= 2'b0;
    d_out <= 9'b0;
    con   <= 8'b0;
end
```

**修复建议**:
将第15行拆分为3个独立的语句：
```verilog
if (~res) begin
    state <= 2'b0;
    d_out <= 9'b0;
    con   <= 8'b0;
end
```

### 检查项39: 变量声明检查（声明时赋值）
**规范要求**: 变量声明是否进行赋值。参考值：否。

**审查结果**: ✅ 通过

**详细说明**:
```verilog
第9行:  reg[1:0] state;
第10行: reg[8:0] d_out;
第11行: reg[7:0] con;
```
- 所有变量声明时都没有赋值
- 符合规范要求

**规范引用**:
根据规范第5条第3款：
> 信号声明时不要赋值。

```verilog

    if (~res) begin
        state <= 2'b0;  // 每行一个赋值 ✅

        con   <= 8'b0;
```
