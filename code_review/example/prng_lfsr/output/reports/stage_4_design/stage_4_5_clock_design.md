**规范要求**: 时钟信号应该是明确的输入，避免从模块内部组合逻辑生成时钟，以保证时钟的稳定性和可预测性。

### 第30项: 时钟用途检查
- **级别**: [R] Rule - 必须遵守
- **检查方法**: 时钟是否被当做数据用于赋值
- **参考**: Verilog_RTL_Coding_Standards-v1.4.3.md 第13条
- **预期值**: 否
- **实际结果**: ✅ 通过

**分析:**
```verilog
always @(posedge clk or posedge res) begin
    if (res) begin
        d <= 4'b1111;
    end else begin
        d[2:0] <= d[3:1];
        d[3]   <= d[3] ^ d[0];
    end
end

assign y = d[0];  // 输出不是时钟
```

时钟`clk`仅用于always块的敏感列表`posedge clk`，未被当做数据用于任何赋值操作，符合规范要求。
