## 逻辑运算详细分析

### 补码转换逻辑

**代码实现** (Line 25):
```verilog
assign comp_8 = data_in[7] ? {data_in[7],~data_in[6:0]+1} : data_in;
```

**逻辑分析**:
- 如果 `data_in[7] == 1`（符号位为1，负数）:
  - 取反加1：`~data_in[6:0] + 1`（绝对值）
  - 拼接符号位：`{data_in[7], ~data_in[6:0]+1}`
  - 结果：补码形式

- 如果 `data_in[7] == 0`（符号位为0，正数）:
  - 直接传递：`data_in`
  - 结果：原值

**问题**: 这种补码转换逻辑实际上是**二进制补码到原码的转换**，而非简单的补码运算。

**验证**:
假设 `data_in = 8'b11111100` (-4的补码表示):
- `data_in[7] = 1` (负数)
- `~data_in[6:0] = ~1111100 = 0000011`
- `~data_in[6:0] + 1 = 0000100` (4)
- `{data_in[7], 0000100} = 10000100` (原码形式的-4)

这与标准补码运算存在差异，需要明确设计意图。
